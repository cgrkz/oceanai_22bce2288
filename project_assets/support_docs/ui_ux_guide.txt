1. COLOR PALETTE
================================================================================

1.1 Primary Colors
------------------
- Primary Green: #4CAF50
  Usage: Primary actions, success states, "Pay Now" button, cart buttons

- Primary Green Hover: #45a049
  Usage: Hover state for primary green elements

1.2 Secondary Colors
--------------------
- Orange: #ff9800
  Usage: Discount code button, promotional elements

- Orange Hover: #e68900
  Usage: Hover state for orange elements

1.3 Status Colors
-----------------
- Success Green: #4CAF50
  Usage: Success messages, valid discount codes, checkmarks

- Error Red: #f44336
  Usage: Error messages, validation errors, remove buttons, discount display

- Warning Yellow Background: #fff3cd
  Usage: Discount code section background

1.4 Neutral Colors
------------------
- Background: #f5f5f5 (light gray)
- Container Background: #ffffff (white)
- Section Background: #f9f9f9 (very light gray)
- Text Primary: #333333 (dark gray)
- Text Secondary: #666666 (medium gray)
- Text Disabled: #999999 (light gray)
- Border Color: #dddddd (light border)
- Border Secondary: #e0e0e0 (lighter border)
- Hover Background: #f5f5f5

4. FORM VALIDATION AND ERROR DISPLAY
================================================================================

4.1 Error Message Requirements
-------------------------------
CRITICAL REQUIREMENT: Form validation errors MUST be displayed in RED text
- Color: #f44336 (red) - MANDATORY
- Font Size: 13px
- Position: Directly below the invalid field
- Display: Initially hidden, shown on validation failure
- Class: "error-message" with "show" class when visible

4.2 Invalid Field Styling
--------------------------
- Border Color: #f44336 (red)
- Border Width: 1px (same as normal)
- Class: "error" added to input element

4.3 Valid Field Styling
------------------------
- Border Color: #4CAF50 (green) on focus
- Box Shadow: 0 0 5px rgba(76, 175, 80, 0.3) on focus

4.4 Real-time Validation
------------------------
- Errors should disappear as user corrects the input
- Validation feedback should be immediate
- Error messages should be specific and helpful

4.5 Required Field Indicators
------------------------------
- Required fields marked with asterisk (*) in label
- Example: "Full Name *", "Email Address *", "Shipping Address *"

11. INTERACTIVE BEHAVIORS
================================================================================

11.1 Hover Effects
------------------
- All buttons: 0.3s background-color transition
- Product cards: 0.2s transform and shadow transition
- Radio options: 0.3s all properties transition

11.2 Focus States
-----------------
- Form inputs: green border (#4CAF50) with subtle shadow
- Remove default outline
- Add custom focus styling for accessibility

11.3 Disabled States
--------------------
- Pay Now button (disabled): #cccccc background, not-allowed cursor
- Disabled condition: Form validation fails or cart is empty

11.4 Loading States
-------------------
- Not implemented in current version
- Future consideration: spinner or skeleton screens

12. RESPONSIVE DESIGN
================================================================================

12.1 Breakpoints
----------------
- Desktop: > 768px
- Tablet: 768px - 481px
- Mobile: < 480px

12.2 Grid Behavior
------------------
- Products grid: auto-fit with minimum 250px columns
- Adapts to screen size automatically

12.3 Mobile Considerations
--------------------------
- Touch targets minimum 44px Ã— 44px
- Larger padding for mobile form inputs
- Stack elements vertically on small screens
- Full-width buttons for better usability

================================================================================

13. ACCESSIBILITY REQUIREMENTS
================================================================================

13.1 Color Contrast
-------------------
- All text meets WCAG AA standards (4.5:1 ratio for normal text)
- Error messages have sufficient contrast with background
- Form labels are clearly readable

13.2 Focus Management
---------------------
- All interactive elements must be keyboard accessible
- Focus states clearly visible
- Logical tab order through form

13.3 Labels and ARIA
--------------------
- All form inputs have associated labels
- Error messages associated with inputs
- Radio buttons properly grouped

13.4 Error Identification
-------------------------
- Errors identified by color AND text
- Error messages are descriptive
- Multiple errors can be displayed simultaneously

================================================================================

14. ANIMATION AND TRANSITIONS
================================================================================

14.1 Standard Transitions
--------------------------
- Button hover: 0.3s background-color
- Product card hover: 0.2s transform
- Radio option: 0.3s all properties

14.2 Smooth Scrolling
---------------------
- Success message scroll: smooth behavior
- Auto-scroll to success message after payment

14.3 Performance
----------------
- Use CSS transitions over JavaScript animations
- Keep animations under 300ms for snappy feel
- Avoid animating expensive properties (width, height)
